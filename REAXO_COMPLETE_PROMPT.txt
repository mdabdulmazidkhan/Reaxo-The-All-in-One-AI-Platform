PROJECT OVERVIEW
================

Build "Reaxo" - a sophisticated multi-AI chat platform that allows users to query multiple AI models simultaneously, compare responses in real-time, and choose the best answer. This is an all-in-one AI comparison platform built with modern web technologies.

The platform supports 20+ AI models from different providers including OpenAI (GPT-5, GPT-4.1, GPT-4o), Anthropic (Claude Sonnet, Claude Haiku), Google (Gemini 3 Pro, Gemini 2.5), xAI (Grok 4), DeepSeek, and many others.

PROJECT TECH STACK
==================

Frontend:
- React 19.2 with TypeScript
- Next.js 16 with App Router (for app pages) + Vite (for src pages)
- Tailwind CSS v4 with custom design tokens
- Shadcn/UI components
- Lucide React icons
- Zod for validation
- React Hook Form for form management
- AI SDK v6 for streaming responses

Backend:
- Next.js API Routes
- Zeabur AI Hub for unified AI model access
- Environment variable: ZEABUR_API_KEY (required for AI model access)

UI Libraries:
- Radix UI (underlying components)
- Sonner for toast notifications
- Recharts for data visualization

Storage & Auth:
- localStorage for session management (user tokens stored as insforge_token)
- Supabase integration (for future database and auth)
- Custom user profile management

DESIGN SYSTEM & COLORS
======================

Color Palette (Dark Theme):
- Background: Pure black (oklch(0 0 0))
- Foreground: Near white (oklch(0.985 0 0))
- Card/Secondary backgrounds: Dark gray (oklch(0.09 0 0) and oklch(0.15 0 0))
- Primary accent: Cyan blue (oklch(0.7 0.15 195))
- Borders: Subtle white with low opacity (borders appear as white/5 to white/20)
- Sidebar: Extra dark (oklch(0.05 0 0))

The entire application uses a sophisticated dark theme with minimal color use. Accent colors are primarily cyan/blue for interactive elements and action buttons, with subtle white accents throughout.

Gradients used:
- Hero section has layered background orbs: indigo-500/20, purple-500/20, and cyan-500/10 with blur effects
- Text gradients: gradient from cyan-400 through violet-400 to pink-400 (used in hero headline)

Typography:
- Font family: Inter (sans-serif) for all text
- Font weights: Regular (400) for body, Medium (500) for labels, Semibold (600) for headings, Bold (700) for main headings
- Line heights: 1.4-1.6 for readability
- Font sizes used: 10px (labels), 11px (model names), 12px (descriptions), 14px (body), 16px (regular text), 18-20px (section text), 24-32px (headings), 48-96px (hero text)

DATA FLOW & STATE MANAGEMENT
============================

1. App State (in page.tsx):
   - user: User | null (authenticated user data)
   - isLoading: boolean (initial auth check)
   - showAuthModal: boolean (auth modal visibility)
   - showProfileModal: boolean (profile modal visibility)
   - isGuestMode: boolean (guest mode flag)
   - token: string | null (auth token)
   - profile: User | null (user profile data)

2. Dashboard State (in dashboard.tsx):
   - enabledModels: Set<string> (selected AI models)
   - sidebarOpen: boolean (mobile sidebar visibility)
   - copiedId: string | null (recently copied response id)
   - conversations: ConversationTurn[] (chat history)
   - input: string (current user input)
   - isLoading: boolean (API call in progress)

3. Conversation Structure:
   - Each turn has: id, prompt, responses[]
   - Each response has: modelId, content, isLoading, error?

4. Model Structure:
   - id: string (unique identifier)
   - name: string (display name)
   - provider: string (provider name)
   - description: string
   - inputCost: string
   - outputCost: string
   - contextWindow: string
   - color: string


API ENDPOINTS
=============

POST /api/chat
- Accepts: { messages: Array<{role, content}>, model: string }
- Returns: Server-sent event stream (text/event-stream)
- Streams JSON chunks with format: data: {choices[0].delta.content}
- Sends [DONE] when complete

GET /api/models
- Returns: List of available models with pricing and details
